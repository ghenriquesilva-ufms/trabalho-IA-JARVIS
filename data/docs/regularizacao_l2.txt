Regularizacao L2 - Resumo

A regularização L2 é uma técnica utilizada em Aprendizado de Máquina para reduzir overfitting e melhorar a capacidade de generalização dos modelos. Ela adiciona uma penalidade aos pesos do modelo durante o treinamento, incentivando valores menores e evitando que o modelo dependa excessivamente de atributos específicos.

A regularização L2 também é conhecida como:

Ridge Regularization;
Ridge Regression;
Weight Decay.

Conceito Fundamental

Em muitos modelos de Machine Learning, especialmente os mais complexos, os pesos podem crescer excessivamente durante o treinamento.

Pesos muito altos podem indicar:

memorização dos dados;
sensibilidade exagerada;
baixa generalização.

A regularização L2 adiciona uma penalidade matemática para desencorajar pesos grandes.

Objetivo da Regularização

A regularização busca:

reduzir overfitting;
melhorar generalização;
estabilizar o modelo;
reduzir sensibilidade ao ruído.

O modelo passa a aprender padrões mais gerais em vez de memorizar dados específicos.

Função de Custo com Regularização L2

Sem regularização:

o modelo minimiza apenas o erro.

Com regularização L2:

o modelo minimiza erro + penalidade dos pesos.

A penalidade L2 é baseada na soma dos quadrados dos pesos.

Fórmula conceitual:

Loss = Erro + λ * Σ(w²)

Onde:

Loss representa a função de custo total;
λ (lambda) controla intensidade da regularização;
w representa os pesos do modelo.

O termo:

Σ(w²)

é chamado de norma L2.

Papel do Lambda

Lambda controla o impacto da regularização.

Lambda pequeno:

pouca regularização;
maior risco de overfitting.

Lambda grande:

forte penalização;
modelo mais simples;
risco de underfitting.

Escolher lambda corretamente é essencial.

Efeito nos Pesos

A regularização L2:

reduz pesos grandes;
distribui importância entre atributos;
evita dependência excessiva de variáveis específicas.

Importante:

os pesos raramente se tornam exatamente zero;
apenas ficam menores.

Diferença entre L1 e L2

Regularização L1:

utiliza soma dos valores absolutos;
pode zerar pesos;
gera modelos mais esparsos.

Regularização L2:

utiliza soma dos quadrados;
reduz pesos suavemente;
mantém todas as variáveis.

L2 tende a produzir modelos:

mais estáveis;
menos agressivos.

Regularização em Regressão

A regularização L2 é muito utilizada em:

regressão linear;
regressão logística.

Na regressão linear:

recebe o nome Ridge Regression.

Ela ajuda quando existem:

muitos atributos;
multicolinearidade;
ruído nos dados.

Multicolinearidade

Multicolinearidade ocorre quando variáveis possuem alta correlação entre si.

Isso pode:

instabilizar os pesos;
dificultar interpretação;
aumentar variância.

A regularização L2 reduz esse problema ao limitar magnitude dos coeficientes.

Uso em Redes Neurais

Em redes neurais profundas:

L2 é amplamente utilizada.

Nesse contexto:

frequentemente chamada de Weight Decay.

Ela ajuda:

a evitar overfitting;
controlar crescimento dos pesos;
melhorar estabilidade do treinamento.

Impacto na Generalização

Generalização é a capacidade do modelo funcionar bem em dados novos.

Sem regularização:

modelo pode decorar o treino.

Com L2:

modelo aprende padrões mais robustos.

Isso melhora desempenho em:

validação;
teste;
produção real.

Vantagens

Principais vantagens:

reduz overfitting;
melhora generalização;
estabiliza treinamento;
reduz sensibilidade ao ruído;
funciona bem em modelos lineares e neurais.

Além disso:

é simples de implementar.

Limitações

Apesar de útil, possui limitações:

não elimina variáveis irrelevantes;
pode causar underfitting se excessiva;
exige ajuste adequado de lambda.

Em alguns problemas:

L1 pode ser mais adequada para seleção de atributos.

Aplicações

Regularização L2 é utilizada em:

regressão linear;
regressão logística;
redes neurais;
visão computacional;
processamento de linguagem natural;
sistemas financeiros;
modelos de recomendação.

Exemplo Intuitivo

Imagine dois modelos:

Modelo A:

pesos extremamente altos;
muito sensível aos dados.

Modelo B:

pesos moderados;
comportamento mais estável.

A regularização L2 favorece:

o Modelo B.

Isso normalmente melhora desempenho em dados desconhecidos.
